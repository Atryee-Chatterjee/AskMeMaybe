Powered by Advanced RAG Architecture

Intelligent document understanding with enterprise-grade accuracy

Retrieval-Augmented Generation

Our RAG pipeline retrieves precise document chunks using semantic search, then generates accurate, grounded answers with source citations. No hallucinations — every response is backed by your actual documents.

PDF Text Extraction

Advanced PyPDF extraction processes documents page-by-page, preserving original formatting, structure, and metadata. Handles complex layouts, tables, and multi-column content seamlessly.

Semantic Chunking & Embeddings

Documents are split into intelligent semantic chunks (1000 chars with 200 overlap) using RecursiveCharacterTextSplitter, then embedded with sentence-transformers (all-MiniLM-L6-v2) for high-quality semantic search.

FAISS Vector Database

High-performance similarity search using FAISS (Facebook AI Similarity Search). Incrementally add documents to existing indexes without reprocessing — perfect for growing document collections.

Multi-PDF Cross-Reference

Upload multiple PDFs and query across all simultaneously. The system retrieves the most relevant chunks from all documents, combining insights from your entire knowledge base.

Source Transparency

Every answer includes exact citations showing which document and page number the information came from. Full transparency into AI responses with clickable source references.

Llama 3.1 Integration

Powered by Meta's Llama 3.1 8B Instruct model via Hugging Face, delivering high-quality, contextually-aware responses with precise temperature control (0.3) for consistent outputs.

Incremental Learning

Upload new documents anytime — the system intelligently updates the FAISS index without rebuilding from scratch. Your knowledge base grows organically over time.

Frequently Asked Questions

Our Retrieval-Augmented Generation (RAG) pipeline first extracts text from your PDFs, splits it into semantic chunks, converts them into vector embeddings using sentence-transformers, and stores them in a FAISS database. When you ask a question, it retrieves the most relevant chunks and feeds them to Llama 3.1 to generate accurate answers with source citations.

Your PDFs are stored locally in the uploads folder and processed into vector embeddings stored in FAISS. The system maintains document metadata including filenames and page numbers for source tracking. Documents remain in your local environment and are only accessible by you.

Our system provides highly accurate responses because the LLM is constrained to answer ONLY from the retrieved document context. Each answer includes source citations (filename and page number) so you can verify the information. The temperature setting of 0.3 ensures consistent, factual responses rather than creative outputs.

Yes! Our recursive text splitter handles large documents by chunking them into 1000-character segments with 200-character overlap, preserving context between chunks. The FAISS vector database efficiently stores and searches across hundreds of pages.

We use sentence-transformers/all-MiniLM-L6-v2, a lightweight yet powerful embedding model that efficiently converts document chunks into 384-dimensional vectors. It balances speed and accuracy perfectly for semantic similarity search across document collections.

Absolutely! The system supports incremental updates. When you upload new PDFs, the text extraction and chunking process runs again, and new vectors are added to the existing FAISS index. No need to reprocess previous documents — your knowledge base grows seamlessly.

The backend runs locally with Flask and FAISS. However, you need internet access for the Hugging Face Llama 3.1 model API calls. For offline use, you could switch to a local LLM like Ollama or GPT4All.

During PDF processing, we track metadata (filename and page number) for each text chunk. When the similarity search retrieves relevant chunks, we extract this metadata and display it with the answer. Duplicate sources are automatically deduplicated, so you see each unique citation only once.
